# Implementation Phases Reference

This guide breaks the Nourish roadmap into implementation phases.

### Phase 1: Foundations

- [x] set up backend project structure
- [x] define schema and entities
- [x] create ingredients lookup table
- [x] support manual recipe creation
- [ ] support AI-assisted recipe metadata suggestions
- [x] seed a small curated recipe library

### Phase 2: Fridge foundation

- [x] manual fridge/pantry/freezer item entry
- [x] expiry tracking
- [x] simple “what can I make?” query
- [x] grocery-to-fridge add flow

### Phase 3: Basic weekly planning

- [x] create weeks and slots
- [x] support blank week planning
- [x] support context-aware slot suggestions
- [x] support swap/skip/eating-out behavior
- [x] support approval flow
- [x] generate grocery list

### Phase 4: Smarter planning

- [x] ingredient overlap logic
- [x] expiring-soon prioritization
- [x] nutrition-gap logic
- [x] snack suggestions
- [ ] leftovers/freezer awareness

### Phase 5: Prep sheet intelligence

- [x] step timing tags
- [x] prep-ahead/day-of breakdown
- [x] grouped prep sheets
- [x] simple task grouping by method or parallel group

### Phase 6: Save and rotate

- [x] save a week as template
- [x] load saved week
- [x] rotation support
- [ ] conflict handling for disliked or outdated items

### Phase 7: Auth and household polish

- [ ] real login/register
- [x] household settings persistence
- [ ] onboarding completion checks
- [x] better multi-user groundwork

### Phase 8: Advanced additions

- [ ] receipt scanning
- [ ] richer substitution logic
- [x] better “in my fridge” recipe browsing
- [ ] shared live grocery updates
- [ ] cooking mode / step-by-step kitchen view
